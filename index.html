Java继承

Java

Java继承

  1. 方法覆盖(重写)

    • 注意事项

      1. 子类覆盖父类方法保证权限大于等于父类
      2. 静态方法只能被静态方法覆盖

    • 重写和重载的区别

      1. 重载:只看同名函数的参数列表
      2. 子父类方法要一模一样(除了多态下返回类型可以用子父类兼容)

  2. 继承下的构造函数

    1. 构造函数函数名一定不同,不可能出现覆盖
    2. 子类构造函数默认会调用空参构造函数(super()),如果显示调用了父类构造函数,则不会隐式调用,调用构造器的语句只能在构造器函数的第一行
    3. 因为this构造器调用也要写在第一行,所以如果第一行写了this语句也不会存在隐式调用super语句,其实调用的其他本类构造函数一定调用了父类的构造函数(隐式或者显式)

  3. final关键字

    1. final修饰的类不能被继承
    2. final修饰的函数不能被重写
    3. final修饰的变量值不能改变且必须初始化,既可以修饰成员变量也可以修饰局部变量(名称全大写多个单词间加_),这样可以增强阅读性,推荐使用
    4. 内部类定义在类的局部位置上时,只能访问该局部被final修饰的局部变量

  4. Java方法和变量在继承中的覆盖和隐藏

    1. 父类的实例变量和静态变量能被子类的同名变量隐藏

    2. 父类的静态方法被子类的同名静态方法隐藏

    3. 父类的实例方法被子类的同名实例变量覆盖

      注意:

      • 不能用子类的静态方法隐藏 父类中同样标示（也就是返回值 名字 参数都一样）的实例方法
      • 不能用子类的实例方法覆盖 父类中同样标示的静态方法
      • 变量只会被隐藏 不会被覆盖 ，无论他是实例变量还是静态变量，而且，子类的静态变量可以隐藏 父类的实例变量，子类的实例变量可以隐藏 父类的静态变量
      • 隐藏 和覆盖 的区别在于，子类对象转换成父类对象后，能够访问父类被隐藏 的变量和方法，而不能访问父类被覆盖 的方法

  5. 接口(interface)

    1. public static final 域

    2. public abstract 方法

    3. 接口被类多实现,接口与接口之间存在多继承

    4. 抽象类与接口的不同点

      • 抽象类用于对对象的共性抽取(基本功能)用继承拿到
      • 接口用于对类的功能扩展,使用实现来扩展(扩展功能)

    5. jdk1.8引入的新功能

      • 接口中可以定义静态函数
      • 接口的抽象函数可以通过default关键字为抽象方法提供一种默认实现(写了default就不用abstract了)

Java对象

Java

1. 一些细节知识点

  • 堆内存中都有默认初始化值

  • 成员变量和局部变量的区别

    • 存储位置不同,成员变量在堆内存,局部变量在栈内存
    • 作用域不同

  • if-else与三元运算符的区别

  • 三元运算符一定会有一个返回值,但if-else不需要

  • 第三块内存区域(方法区/数据区/共享区)

    • 用于存放方法代码和static成员变量

  • 构造代码块{}

    • 作用:给对象进行统一初始化,优先于构造函数执行

  • 和构造函数的区别:构造代码块是给所有对象进行统一初始化,而构造函数是给对应的对象进行初始化

  • 静态代码块static{}

    • 作用:给类进行初始化
    • 注意:只有使用到类中的内容才进行类的加载，dataType d = null可不会进行类的加载
    • 随着类的加载而执行且只执行一次

  • 静态的使用注意

    • 随着类的加载而加载,随着类的消失而消失

    • 静态成员是用于对所有对象共享的数据进行存储

    • 静态成员生命周期过长,不要定义过多的静态变量

    • 静态函数中不能使用this,super关键字

  • 什么时候定义静态函数?

    • 当功能内部没有访问到非静态数据,那么该功能可以定义成静态的

  • 由系统生成的默认空参构造函数的权限

    • 一个类中默认会有一个空参构造函数,这个构造函数的权限和所属类的权限相同(public/default)

  • 封装(Encapsulation)

    • 好处

      • 内部变化不会显式出现
      • 便于使用
      • 提高重用性
      • 提高安全性

    • 注意:private仅仅是封装的一种的表现形式,不要认为封装就是private

  • this关键字

    • 代表一个调用本函数的本类对象,可以用于构造函数之间互相调用
    • this语句(就是指构造函数之间互相调用)必须放在构造函数的第一行(只能调用一次),因为初始化动作要最先发生,且只有一次。

  • eclipse传入命令行参数方法

    • Run as按钮下有Run Configurations可以配置

  • 对象的加载过程(无继承)

    • 加载类(加载方法区的方法代码和静态成员)->执行静态成员默认初始化->执行静态成员显式初始化->static代码块执行->堆中开辟对象空间->执行成员变量默认初始化->执行成员变量显式初始化->执行构造代码块->执行对应的构造函数->将内存地址返回

  • 枚举类型(enum)

    • 定义:enum Result{SUCCESS,FAIL;}
    • 默认继承java.lang.Enum类实现Comparable和Serializable接口,里面定义的都是实例对象且为public static final
    • 判断枚举类型相等可以直接用==
    • java.lang.Enum类有一些方法可以直接用

  • 常用修饰符

    • public static final void

  • 继承的小知识

    • 对父类私有的方法不视为重写,因为对子类不可见

2.包与访问权限

  • Java中有四种访问权限(public/protected/default/private)

4.其他的细节

  1. 浏览器可以直接访问WebContent中的文件
    但是其中WEB-INF中的文件无法通过客户端直接进行访问,可以通过请求转发来访问

  2. Jsp中的编码问题

    1. pageEncoding:<%@ page pageEncoding="UTF-8"%>

      jsp页面编码: jsp文件本身的编码

    2. contentType: <%@ page contentType="text/html; charset=UTF-8"%>

      web页面显示编码:jsp的输出流在浏览器中显示的编码

    3. html页面charset:<META http-equiv="Content-Type" content="text/html; charset=UTF-8">

      web页面输入编码: 输入框输入的字体编码 {这个编码存疑,测试不出}

  3. 重定向与请求转发的区别

    • 重定向地址栏改变,转发不改变
    • 请求转发保留第一次请求时的数据,而重定向不会
    • 重定向请求2次响应2次,转发请求1次响应1次
    • 转发在服务器端跳转,重定向在客户端跳转

Java集合

Java

集合

问题

  1. 既然有数组了为什么还要有集合

    答:因为数组定长且只能存储一种类型的数据

  2. 为什么会有这么多的容器

    答:内部的数据存储结构不同

集合继承实现图

Collection接口

  • List和Set接口的父接口
  • List和Set接口的区别
    • List:元素可以重复,有索引
    • Set:元素无序且不能重复
    • 此处指的顺序是指存入和取出顺序不同
  • 要记得特有方法
    • obj.isEmpty();
    • obj.size();
    • obj.contains(Object o);
    • obj.retainAll(Collection<?> c):求交集并改变调用的集合
    • obj.add(E e)
    • obj.remove(Object o)
  • 迭代器(Iterator接口)
    • Iterator obj.iterator();
    • 每一个容器中都定义了一个迭代器用于操作容器本身

List接口

  • 集合的常用方法

    • 增:add(）
    • 删:remove()
    • 改:set()
    • 查:get(),subList()[对其操作会反馈到原集合],ListIterator()
    • 获取下标:indexOf()

  • List集合中引入了ListIterator迭代器

    • 因为原来的Iterator接口只提供了remove和next两种操作,为了避免并发修改异常在该迭代器中提供了增删改查操作,以及与next相反的previous操作
    • java.util.ConcurrentModificationException 并发修改异常 出现在使用迭代器的next的同时使用了集合的修改方法引发的异常

  • 引出的子类对象

    • ArrayList:数组结构,查找快增删慢,jdk1.2出现用于替代Vector,空间增长为50%增长,线程不同步

    • LinkedList:链表结构,查找慢,增删快,线程不同步

      • 特有方法(First/Last)
        • removeFirst/Last()[集合为空会出异常] —–pollFirst/Last()[jdk1.6]集合为空返回null
        • addFirst/Last()void —–offerFirst/Last()[jdk1.6]Boolean
        • getFirst/Last()[集合为空会出异常] —–peekFirst/Last()[jdk1.6]集合为空返回null

    • Vector:数组结构,jdk1.0出现,线程同步,空间100%增长

      • 特有方法(Element)
        • Enumeration obj.elements();
        • Enumeration 枚举类型 其实就是Iterator前身,用法相同

  • 判断元素是否相同根据equals方法来确定

Set接口

  • 基本方法于Collection接口相同

  • 引出的子类对象

    • HashSet:哈希表结构,线程不同步,避免哈希冲突的方式类似于链地址法
      • 存储时先比较hashCode方法值是否相同,不同则存储,相同则比较equals判断,如果仍然相同,认为是统一对象,否则在hachCode值存储的位置下扩展链。
      • 比较对象是否相同先比较hashCode再比较equals,只有两者都相同才认为两对象相等。

  • TreeSet:红黑树结构(属于二叉排序树),线程不同步,有两种比较方式

    • 存储类实现java.lang.Comparable接口,实现compareTo方法,称为自然顺序,很多类都实现了(String,Integer)
    • 构造器类实现java.util.Comparator接口,实现compare方法,称为构造器,将构造器传入集合构造方法,实现排序
    • 如果上述两种方式都实现了以构造器为主
    • TreeSet标识元素相等的条件为compareTo方法或者compare方法的返回值(0)

Java泛型(jdk1.5)

  • 虽然集合可以存储任意对象带来了方便,但是因为类型不明确导致强制类型转换操作过多,且代码不明确,引入了泛型,泛型只能用于引用数据类型
  • ArrayList <String> list = new ArrayList<String>();
  • 泛型可以定义在类上,方法上,接口上
  • 注意:在静态方法中是不能访问定义在类上的泛型的,因为只有在定义对象时才能确定类上的泛型,而在此之前静态方法已经出现
  • this.<String>show();
  • 调用定义了泛型的方法为:在方法名前加<具体数据类型>,但是可以通过形式参数来推断,所以大多数时候可以省略
  • <?>代表任意类型,?代表统配符,之后又提出了泛型限定<? extends E>,<? super E>,前者在Collection中有应用,后者在Comparator接口中有应用。泛型限定用于对？的功能进行进一步的限定。
  • ArrayList <Object> list = new ArrayList<String>();
  • 上面这样的代码会出错,泛型并不支持这种多态,这样会导致声明为String的列表中有可能会存放其他Object的子类对象造成类型转换异常。

Map接口

  • 虽然Map接口并没有实现Collection接口但是仍然属于集合框架

  • Map<K,V>,用于存储键值对,其中键唯一,值不唯一

  • 常用方法

    • 1.添加:

      • V put(K key, V value):存相同的键会替代前键值对并返回原来的值,如果没有相同的键会返回null(与Set集合不同要注意)

    • 2.删除

      • clear()

      • remove(Object key)

    • 3.判断

      • containsKey(Object key)

      • isEmpty()

    • 4.获取

      • get(Object key)

      • size()

      • Set<V> values():获取值的Set集合

      • Map.Entry<K, V>[内部的静态接口] entrySet()

      • Set<K> keySet()

  • 常用的子类(所用比较的方法都是对键操作,与Set集合类似)

  • Hashtable

    • 底层是哈希表,不能存入null键或null值 jdk1.0 线程同步

  • HashMap

    • 底层也是哈希表,允许使用null键或null值 jdk1.2 线程不同步 效率高

  • TreeMap

    • 底层二叉树排序树结构,线程不同步,给map集合中的键进行排序

  • Set底层就是用了Map的相关实现方法

Utils接口

  • Collections类(里面全是静态方法)

    • 常用方法

    • 排序

      • <T extends Comparable<? super T>> void sort(List<T> list)
      • void sort(List<T> list,Comparator<? super T> c)

    • 最大最小

      • <T extends Object & Comparable<? super T>> T max(Collection<? extends T> coll)
      • <T> T max(Collection<? extends T> coll,Comparator<? super T> comp)
      • 为什么只有List有比较和最大值/最小值 因为Set有TreeSet,Map有TreeMap且它们都有相对应的First/Last方法

    • 查找

      • int binarySearch(List<? extends Comparable<? super T>> list,T key)
      • <T> int binarySearch(List<? extends T> list,T key,Comparator<? super T> c)
      • 为什么只有List有二分查找,因为只有它有脚标

    • 填充/替换/反转/交换/随机排序

      • <T> void fill(List<? super T> list,T obj)
      • boolean replaceAll(List<T> list,T oldVal,T newVal)
      • void reverse(List<?> list)
      • void swap(List<?> list,int i,int j)
      • void shuffle(List<?> list)
      • 如果要对列表的一部分填充的话可以用list中的subList对原列表进行填充

    • 获取逆向的比较器

      • <T> Comparator<T> reverseOrder():获取自然顺序逆序的比较器
      • <T> Comparator<T> reverseOrder(Comparator<T> cmp):获取指定比较器的逆向比较器

    • 获取线程同步的集合

      • <T> List<T> synchronizedList(List<T> list)
      • <T> Set<T> synchronizedSet(Set<T> s)
      • <K,V> Map<K,V> synchronizedMap(Map<K,V> m)

数组与集合的转换

  • Arrays类提供了许多数组操作的方法[数组转集合]

    • <T> List <T> asList(T... a):数组转换为List集合,

  • 原因在于可以方便去判断元素包含于集合[使用集合的方法来操作数组]

  • 生成的集合是定长的且改变会反馈到数组

  • 如果数组中的元素是基本数据类型,则会直接把数组本身存入到集合中

  • Collection类中的toArray方法[集合变数组]

    • Object[] toArray()

    • <T> T[] toArray(T[] a)

  • arrayList.toArray(new String[arrayList.size()])

    • 注意如果指定类型的数组长度小于集合的长度,那么方法内部会创建一个新的数组。长度为集合的长度

    • 如果如果指定类型的数组长度大于集合的长度,那么方法内部会使用传入的数组。多余的空间设置为null

    • 所以尽量指定一个大小刚好的数组,可以节省空间

    • 一个方法 可以有可变参数和普通参数，但是可变参数必须放到参数列表末尾；
    • 一个方法 有且只能有一个可变参数;

  3. 静态导入

    • import static java.util.Arrays.*
    • 导入一个类/接口的静态成员，之后调用可以省略类名/接口名
  • 如果出现重名的函数,则要恢复会原来的调用方式
    • 主要用于对都是静态成员的工具类的引入,简化书写。比如Collections,Arrays

Java日期

Java日期格式

1.JDK7之前的方式

  1. java.util.Date
    • new Date():以当前时间为准
    • new Date(Long date):以毫秒时间创建对象
    • object.getTime():获取毫秒数
  • form(Instant instant):通过Instant对象创建Date对象-static
    • object.toInstant():返回对应date的Instant对象

  1. java.sql.Date

    • 与数据库的Date类型对接的类型.是java.util.Date的子类
    • new Date(Long date):以毫秒时间创建对象
    • 通过毫秒数与java.util.Date进行转换

  2. java.text.SimpleDateFormat

    • 格式化日期类
    • new SimpleDateFormat():默认格式
    • new SimpleDateFormat(String pattern):通过指定格式来格式化日期
    • object.format(Date date):格式化Date返回一个字符串
    • object.parse(String source):解析一个字符串返回Date对象
    • 注意:Date指的都是java.util.Date

  3. java.util.Calendar

    • 日历类
    • Calendar.getInstance():创建一个Calendar对象
    • object.get(int field):获取类中定义的常量字段
    • object.set(int field, int value):设置类中定义的常量字段,改变对象
    • object.add(int field, int amount):增加类中定义的常量字段,改变对象
    • object.getTime():返回对应的Date对象
    • object.setTime(Date date):通过对应的Date对象,设置Calendar,改变对象
    • 注意:Date指的都是java.util.Date
  • 月份从0(一月)开始，星期从1(周日)开始
    • 对象可变
  1. 因为上述的处理线程不同步,且对日期的处理并不完善在JDK8中有提出了新方式

2.JDK8的新方式

  1. java.time.LocalDate,java.time.LocalTime,java.time.LocalDateTime

    • 都是线程安全且不可变对象
    • 通用方法:
    • now():创建当前时间的对象-static
    • of(xxxxxx):创建指定时间的对象-static
    • object.getXXX():获取指定的信息
    • object.withXXX(xxxx)[等于set]:设置指定信息并返回相应对象
    • object.plusXXX(xxxx):给对应信息加上指定数值并返回相应对象
    • object.minusXXX(xxxx):给对应信息减去指定数值并返回相应对象
    • parse(CharSequence text, DateTimeFormatter formatter):解析指定字符串并返回相应对象-static
    • obj.format(DateTimeFormatter formatter):返回格式化对象

  2. java.time.Instant

    • 时间线上的瞬时点
    • 注意如果不设置时区偏移,则返回的时间是本初子午线上的时间
    • now():创建当前时间的对象-static
    • ofEpochMill(long epochMill):设置指定毫秒数的对象-static
    • object.toEpochMill():返回毫秒数
    • obj.atOffset(ZoneOffset offset):设置时区的偏移如ZoneOffset.ofHours(8)

  3. java.time.format.DateTimeFormatter

    • ofPattern(String pattern):创建对应格式的DateTimeFormatter对象-static
    • 注意在DateTimeFormatter格式字符串中小时用H表示

  4. 各种转换

    代码来自博客 Java日期转换

  5. 遇到的一个问题

    • LocalDateTime dateTime = LocalDateTime.parse("2020-02-02 03:10:10", DateTimeFormatter.ofPattern("yyyy-MM-dd hh:mm:ss"));

    • 上面的代码使用H可以运行但是换成h就无法运行

1.对java.lang.Class类的理解

类的加载过程

程序经过javac.exe命令以后会生成一个或多个字节码文件(.class结尾) 接着我们使用java.exe命令对某个字节码文件进行解释运行。相当于将某个字节码文件加载到内存中,这个过程称为类的加载,加载到内存中的类称作运行时类,作为Class的一个实例。换句话说，Class的一个实例就对应一个运行时类。

  • new T(xxx).getClass();

  • Class.forName(完整类名);
