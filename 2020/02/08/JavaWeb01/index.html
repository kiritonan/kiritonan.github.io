1.tomcat端口更改

tomcat默认端口为8080,在conf/server.xml中70行左右connector标签的port值修改

2.状态码

200:一切正常
3xx:重定向
403:权限问题
404:资源不存在
4xx:客户端错误
5xx:服务器端错误
500:服务器端错误

3.把项目放到其他路径配置

1.在conf/server.xml的Host标签中配置
<Context docBase = "表示实际转定位到的路径" path = "表示网页要访问的路径" />
path:可以写相对(相对于webapps因为Host在localhost中首先找的就是webapps),写绝对也没事
例如 /jspTest
docBase:表示实际转定位到的路径
D:\apache-tomcat-8.5.50\jspTest
写完重启服务器
另一种方法
2.conf\Catalina\localhost 新建项目名.xml文件
在里面写<Context docBase = "实际项目路径"/>
就行

4.其他的细节

  1. 浏览器可以直接访问WebContent中的文件
    但是其中WEB-INF中的文件无法通过客户端直接进行访问,可以通过请求转发来访问

  2. Jsp中的编码问题

    1. pageEncoding:<%@ page pageEncoding="UTF-8"%>

      jsp页面编码: jsp文件本身的编码

    2. contentType: <%@ page contentType="text/html; charset=UTF-8"%>

      web页面显示编码:jsp的输出流在浏览器中显示的编码

    3. html页面charset:<META http-equiv="Content-Type" content="text/html; charset=UTF-8">

      web页面输入编码: 输入框输入的字体编码 {这个编码存疑,测试不出}

  3. 重定向与请求转发的区别

    • 重定向地址栏改变,转发不改变
    • 请求转发保留第一次请求时的数据,而重定向不会
    • 重定向请求2次响应2次,转发请求1次响应1次
    • 转发在服务器端跳转,重定向在客户端跳转

5.jSP的页面元素

  1. jsp脚本标识

    • <% 局部变量/Java语句 %> JSP表达式 本页面有效
    • <%! 成员变量/成员方法 %> 声明标识
    • <%= 变量/有返回值的函数或表达式%>不用分号,会调用对象的toString方法–脚本程序

  2. jsp指令标识(<%@指令名称 属性1=”” 属性2=””%>)

    • page指令(页面指令)
      • <%@ page language=”java” contentType=”text/html; charset=UTF-8” pageEncoding=”UTF-8” import=”java.util.Date”%>
      • 主要是import指令引入java类,多个包之间用,分隔

  3. 注释

    • Html注释
    • jsp注释:<%– –%>
    • java程序中的注释
    • 其中第2,3不会被用户在浏览器中查看源码时看见

6.九大内置对象（一些不常用就不写了）

  1. out –输出流

  2. request –请求对象,客户端向服务器端发送的请求

    • 常用方法

      • String getParameter(String name)

      • String[]getParameterValues(String name):根据name,返回多个值[用于多选框]

      • void setCharacterEncoding(String name):设置post请求编码(如果不设置服务器会使用默认的,如果是tomcat7以前是iso-885901,8及以后都是utf-8)

      • getRequestDispatcher(String pagePath).forward(request.response);:请求转发

      • getServletContext();:获取项目的ServletContext对象

      • 用get/post方式提交的乱码解决问题方式

        • get:

          1. 统一乱码变量的编码(不推荐)

            • 改变字符串变量的编码方式
            • new String(str.getBytes(“旧编码”),”新编码”);

          2. 修改server.xml,一次性修改tomcat默认get提交方式的编码

            • 在修改端口的标签中加入URIEncoding = “所需编码”

        • post:

          1. request.setCharacterEncoding(“所需编码”)

  3. response:响应对象

    • 常用方法
      • void addCookie(Cookie cookie):服务端向客户端增加Cookie对象
      • void sendRedirect(String path)[throws IOException]:页面重定向
      • void setContentType(String type):设置相应类型和编码
    • 更改response的out使用编码和浏览器显示out的输出解析编码
      • response.setCharacterEncoding(“utf-8”);Sets the character encoding (MIME charset) of the response being sent to the client
      • response.setContentType(“text/html;charset=UTF-8”);Sets the content type of the response being sent to the client

  4. Cookie(客户端,不是内置对象)伪

    • Cookie是服务端生成的,在发给客户端保存,包含键值对[javax.servlet.http.Cookie]

    • 常用方法

      • public Cookie(String name,String value);
      • String getName();
      • String getValue();
      • void setMaxAge(int expiry);最大有效期

    • 服务端准备客户端Cookie,重定向/转发时返回客户端时添加

      • response.addCookie(Cookie cookie);

    • 客户端获取Cookie:

      • request.getCookies();—没有拿特定Cookie的方法

    • 服务端会自动产生一个名称为JSEESSIONID的Cookie

  5. Session

    • 生成方式

      • 客户端第一次请求时,(匹配失败则)服务器端生成一个seesion,会自带一个唯一的sessionID,随后会生成一个JSEESSIONID的Cookie值相同,随响应传递给客户端,客户端与服务端的会话的对应是由JSEESSIONID的值和sessionID对应来实现的.客户端第n次请求时,不需要再创建session了,用JSEESSIONID找到session,如果找到,则匹配成功,说明以前访问过。

    • 和Cookie的不同

      • 存放位置不同
      • 安全性不同(Session保存在服务器端更安全些)
      • 保存的内容(Cookie内容只能存String而Session可以存任意对象

    • 方法

      • String getId()–获取sessionID

      • boolean isNew():判断是否是第一次访问

      • void invalidate():使session失效

      • void setMaxInactiveInterval(s):设置最大有效非活动时间

      • int getMaxInactiveInterval():获取最大有效非活动时间

int getMaxInactiveInterval():获取最大有效非活动时间

  1. application—全局对象

    • 方法

      • String getContextPath():项目的虚拟路径 网页访问的路径

      • String getRealPath(String name):获取应用任意虚拟路径对应的绝对文件路径

7.四大范围对象

  1. PageContext :当前页面有效
  2. request : 同一次请求有效
  3. session : 同一次会话(默认浏览器打开到关闭算一次会话)有效
  4. application : 同一个项目有效

  • 共有方法

    • Object getAttribute(String name)
    • void setAttribute(String name,Object obj)–可以增加也可以修改
    • void removeAttribute(String name)

  • 使用注意

    • 以上范围对象,尽量使用最小的范围对象,因为范围大的对象开销也大

      3. 选取字段名获取的方法

  • 一个小知识

    • Sql中使用--代表注释可以用于SQL注入中

  • 读入什么类型,读出就什么类型

  • 处理CLOB/BLOB 大文本数据

    • CLOB和BLOB的区别

      • CLOB使用CHAR来保存数据。 　如：保存XML文档。MySql中是Text
      • BLOB就是使用二进制保存数据。 　如：保存位图。

    • 写入CLOB/Text可以使用
      preparedStatement的prep.setCharacterStream(2, new FileReader("D:\\诛仙.txt"));
      来进行写入字符输入流

    • 写入Blob使用
      prep.setBinaryStream(2, new FileInputStream("D:\\对象的序列化.avi"));来写入字节输入流

    • 在写入的时候可能会遇到可能会遇到packet(包)太大的问题

      • 改Mysql数据库配置的max_allowed_packet的值就行

    10.El表达式(Expression Language)

    • 简介

      • 用于替换JSP页面的Java代码

    • 对于不存在的属性和为null的属性显示为空

      • EL中的字符串只要是引号就行
      • 注意:通过EL表达式取自定义对象的属性,它只会去调用get/set方法

    • 用法

      1. 基本格式:${}
      2. 取对象属性操作符(map中的键的值):[]或. []可以放特殊字符也可以使用变量,可以访问数组元素,如果使用名字要加引号
      3. 运算符:加减乘除取余,empty前缀运算符(判断值为null或者就不存在),关系运算符,逻辑运算符,三元运算符
      4. el中不能用+来进行字符串连接

    • EL中的常用隐式对象

      1. 作用域对象

        • applicationScope/sessionScope/requestScope/pageScope
        • 如果不写域对象的变量,则会按照作用域的大小从小到大进行搜索

      2. 参数访问对象

        • 获取请求中的属性和属性列表
        • param==request.getParameter();
        • paramValues==request.getParameterValues()

      3. Jsp隐式对象

        • PageContext
        • 在jsp中可以通过此对象获取到一切其他的内置对象在el中也可以访问到。通过.或[]

10.JSTL标准标签库

  1. 导包

    • jstl.jar
    • standard.jar

  2. 使用taglib指令标识

    • <%taglib uri="http://java.sun.com/jsp/jstl/xxx" prefix=""%>

  3. JSTL有5种标签库

    1. 核心标签库

      1. 表达式

      • <c:set/>:用于定义变量和修改变量

        • <c:set scope=”request” var=”jstl” value=”test”/>
        • <c:set value=”yang” target=”${requestScope.uStudent}” property=”name”/>
        • 其中target可以是JavaBean也可以是Map对象

      • <c:out/>:用于替代<%= %>

        • 更加强大的输出
        • 对于不存在的属性和为null的属性显示为空
        • <c:out value=”${sessionScope.map}” default=”111” escapeXml=”true”/>

      • <c: remove />:用于移除指定Jsp作用域中指定的变量

        • <c:remove var=”a” scope=”request”/>

      1. 流程控制

        • <c:if></c:if>:代替if

          • <c:if test=”${10>7}”>真</c:if>

        • <c:choose></c:choose>/<c:when></c:when>/<c:otherwise></c:otherwise>:实现if-else/switch(其中可以不写otherwise)

          1. 循环控制

            • <c:forEach></c:forEach>

              • 即可以以实现普通循环也可以实现foreach
              • varStatus用el使用其属性值

12.Servlet

  1. 特点

    • extends javax.servlet.http.HttpServlet
    • 重写doGet()和doPost()方法
    • 单例模式,线程不同步

  2. 配置方法

    • servlet2.5中

  • servlet3.0及以上注解
    • @WebServlet(“url-pattern”):写在servlet类上

  1. Servlet生命周期(5个阶段)

    1. 加载–Servlet容器自动控制

    2. 初始化–init(),该方法再Servlet被加载并实例化以后执行,初始化只执行一次,默认为第一次访问执行,可以修改

      • Servlet初始化时机的配置

      servlet3.0及以上注解

    3. 服务–service()[HttpServlet]

    4. 销毁–destroy(),Servlet被系统回收时执行() 关闭服务器

    5. 卸载–Servlet容器自动控制

  2. 自定义Servlet的继承层次

    • extends HttpServlet
    • HttpServlet extends GenericServlet [定义doGet/doPost]
    • GenericServlet implements Servlet, ServletConfig, java.io.Serializable

  3. Servlet API

    • ServletConfig

      • public ServletContext getServletContext(); 获取Application对象

      • public String getInitParameter(String name); 获取当前Servlet的初始化参数

        配置初始化参数

        注解形式

        • @WebServlet(value="/WelcomeServlet",loadOnStartup = 1,initParams = {@WebInitParam(name = "servletParm",value = "current")})

    • ServletContext中的常见方法

      • 除开Application对象中说到的

      • public String getInitParameter(String name); 获取当前Web应用的初始化参数

      • 全局配置参数

  4. 在Servlet中拿到jsp的内置对象

    • Session: request.getSession();
    • out: response.getWriter();
    • Application: request.getServletContext();

13.过滤器（Filter）

  1. 生命周期:服务器开启时初始化,关闭时销毁

  2. 配置和Servlet相似

    注解方式

    • filter映射中可以使用通配符*
    • dispatcher方式(<dispatcher></dispatcher>)
      • REQUEST:只拦截Http请求
      • FORWARD:只拦截请求转发的请求
      • 还有两种不常用INCLUDE/ERROR

  3. 自定义Filter

    • 实现Filter接口并实现方法
    • 通过doFilter执行过滤操作
    • 通过其中chain.doFilter(request, response);传递

  4. 不同过滤器过滤同一资源的执行顺序

    • 通过web.xml中的filter-mapping的顺序进行执行
    • 注解方式并未提供相关顺序的规定,可以用xml方式配置

14.监听器(Listener)

  1. 配置

    • @WebListener
    • <listener><listener-class>xxx</listener-class></listener>
    • 通过实现不同的接口来实现监听操作,通过参数(不同Event对象)来获取监听的对象以此来对其进行操作。

  2. 监听对象

    • request:ServletRequest
    • session:HttpSession
    • application:ServletContext

  3. 第一类监听器

    • 由服务器来创建对象,我们只需定义

    • 监听对象的创建和销毁

      • ServletContextListener
      • HttpSessionListener
      • ServletRequestLinseter

    • 监听对象中属性的变更

      • ServletContextAttributeListener
      • HttpSessionAttributeListener
      • ServletRequestAttributeLinseter

  4. 第二类监听器

    • 我们要自己创建对象来进行监听

    • 这两个监视器都不需要进行配置

    • Session中监听器对象的绑定/解绑

      • HttpBindingListener
      • 绑定指的就是设置Session的属性,定义和修改都算
      • 解绑就指的是移除Session中的监听器对象

    • Session的钝化/活化

      • HttpSessionActivationListener

      • 钝化和活化(其实就是对象的序列化和反序列化所以存放到硬盘上的自定义对象要实现Serializable)

      • 如何钝化/活化

        • 钝化(生成的文件名就是SessionId)

        • 在tomcat的/conf/context.xml里配置

        • 活化

        • 活化自动进行,访问服务器会去硬盘中寻找符合的session,不符合就重新创建,符合就活化该session并使用
