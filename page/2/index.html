Java对象

Java,Java存疑

1. 一些对象细节知识点

  • 堆内存中都有默认初始化值

  • 成员变量和局部变量的区别

    • 存储位置不同,成员变量在堆内存,局部变量在栈内存
    • 作用域不同

  • if-else与三元运算符的区别

  • 三元运算符一定会有一个返回值,但if-else不需要

  • 第三块内存区域(方法区/数据区/共享区)

    • 用于存放方法代码和static成员变量

  • 构造代码块{}

    • 作用:给对象进行统一初始化,优先于构造函数执行

  • 和构造函数的区别:构造代码块是给所有对象进行统一初始化,而构造函数是给对应的对象进行初始化

  • 静态代码块static{}

    • 作用:给类进行初始化
    • 注意:只有使用到类中的内容才进行类的加载，dataType d = null可不会进行类的加载
    • 随着类的加载而执行且只执行一次

  • 静态的使用注意

    • 随着类的加载而加载,随着类的消失而消失

    • 静态成员是用于对所有对象共享的数据进行存储

    • 静态成员生命周期过长,不要定义过多的静态变量

    • 静态函数中不能使用this,super关键字

  • 什么时候定义静态函数?

    • 当功能内部没有访问到非静态数据,那么该功能可以定义成静态的

  • 由系统生成的默认空参构造函数的权限

    • 一个类中默认会有一个空参构造函数,这个构造函数的权限和所属类的权限相同(public/default)

  • 封装(Encapsulation)

    • 好处

      • 内部变化不会显式出现
      • 便于使用
      • 提高重用性
      • 提高安全性

    • 注意:private仅仅是封装的一种的表现形式,不要认为封装就是private

  • this关键字

    • 代表一个调用本函数的本类对象,可以用于构造函数之间互相调用
    • this语句(就是指构造函数之间互相调用)必须放在构造函数的第一行(只能调用一次),因为初始化动作要最先发生,且只有一次。

  • eclipse传入命令行参数方法

    • Run as按钮下有Run Configurations可以配置

  • 对象的加载过程(无继承)

    • 加载类(加载方法区的方法代码和静态成员)->执行静态成员默认初始化->执行静态成员显式初始化->static代码块执行->堆中开辟对象空间->执行成员变量默认初始化->执行成员变量显式初始化->执行构造代码块->执行对应的构造函数->将内存地址返回

  • 枚举类型(enum)

    • 定义:enum Result{SUCCESS,FAIL;}
    • 默认继承java.lang.Enum类实现Comparable和Serializable接口,里面定义的都是实例对象且为public static final
    • 判断枚举类型相等可以直接用==
    • java.lang.Enum类有一些方法可以直接用

  • 常用修饰符

    • public static final void

  • 继承的小知识

    • 对父类私有的方法不视为重写,因为对子类不可见

2.包与访问权限

  • Java中有四种访问权限(public/protected/default/private)

3.内部类

  1. 常规内部类

    • 访问规则:

      1. 内部类可以直接访问外围类中的所有成员

        • 因为内部类中持有了一个外围类中的引用 外部类.this

      2. 外围类访问内部类,必须建立对象(正常创建就行)

      3. 内部类中所有的静态域都必须是final类型,因为我们希望一个静态域只有一个实例。这个域只能是基本数据类型

      4. Java规范并不希望定义静态方法在常规内部类中

    • 在外部类中创建常规内部类的对象的格式

      • Outer.Inner inner = new Outer().new Inner();

  2. 静态内部类

    • 访问规则

      1. 除了没有生成对外围类的引用特权(Outer.this)外,其他基本一样

      2. 静态内部类可以定义静态域和静态方法

    • 在外部类中创建静态内部类的对象的格式

      • Outer.Inner inner = new Outer.Inner();

  3. 内部类定义在局部时

    1. 不能被成员修饰符修饰

    2. 可以直接访问外围类中的成员,因为仍然持有外部类中的引用

    3. 只能访问该局部被final修饰的局部变量

  4. 匿名内部类

    1. 匿名内部类是内部类的简写格式

    2. 定义匿名内部类的前提

      • 继承一个类/实现一个接口
      • 格式: new 类/接口(构造器参数){}:如果是抽象类或者接口将构造器参数清空即可。

    3. 匿名内部类就是一个匿名子类对象

  5. 内部类中的定义规则

  • 当描述事物时,事物内部还有事物,将该事务用内部类实现, 因为内部类可以访问外围类的成员。

  1. 小tip

    • 声明在接口中的内部类自动声明为public static

4.其他的细节

  1. 浏览器可以直接访问WebContent中的文件
    但是其中WEB-INF中的文件无法通过客户端直接进行访问,可以通过请求转发来访问

  2. Jsp中的编码问题

    1. pageEncoding:<%@ page pageEncoding="UTF-8"%>

      jsp页面编码: jsp文件本身的编码

    2. contentType: <%@ page contentType="text/html; charset=UTF-8"%>

      web页面显示编码:jsp的输出流在浏览器中显示的编码

    3. html页面charset:<META http-equiv="Content-Type" content="text/html; charset=UTF-8">

      web页面输入编码: 输入框输入的字体编码 {这个编码存疑,测试不出}

  3. 重定向与请求转发的区别

    • 重定向地址栏改变,转发不改变
    • 请求转发保留第一次请求时的数据,而重定向不会
    • 重定向请求2次响应2次,转发请求1次响应1次
    • 转发在服务器端跳转,重定向在客户端跳转

6.九大内置对象（一些不常用就不写了）

  1. out –输出流

  2. request –请求对象,客户端向服务器端发送的请求

    • 常用方法

      • String getParameter(String name)

      • String[]getParameterValues(String name):根据name,返回多个值[用于多选框]

      • void setCharacterEncoding(String name):设置post请求编码(如果不设置服务器会使用默认的,如果是tomcat7以前是iso-885901,8及以后都是utf-8)

      • getRequestDispatcher(String pagePath).forward(request.response);:请求转发

      • getServletContext();:获取项目的ServletContext对象

      • 用get/post方式提交的乱码解决问题方式

        • get:

          1. 统一乱码变量的编码(不推荐)

            • 改变字符串变量的编码方式
            • new String(str.getBytes(“旧编码”),”新编码”);

          2. 修改server.xml,一次性修改tomcat默认get提交方式的编码

            • 在修改端口的标签中加入URIEncoding = “所需编码”

        • post:

          1. request.setCharacterEncoding(“所需编码”)

  3. response:响应对象

    • 常用方法
      • void addCookie(Cookie cookie):服务端向客户端增加Cookie对象
      • void sendRedirect(String path)[throws IOException]:页面重定向
      • void setContentType(String type):设置相应类型和编码
    • 更改response的out使用编码和浏览器显示out的输出解析编码
      • response.setCharacterEncoding(“utf-8”);Sets the character encoding (MIME charset) of the response being sent to the client
      • response.setContentType(“text/html;charset=UTF-8”);Sets the content type of the response being sent to the client

  4. Cookie(客户端,不是内置对象)伪

    • Cookie是服务端生成的,在发给客户端保存,包含键值对[javax.servlet.http.Cookie]

    • 常用方法

      • public Cookie(String name,String value);
      • String getName();
      • String getValue();
      • void setMaxAge(int expiry);最大有效期

    • 服务端准备客户端Cookie,重定向/转发时返回客户端时添加

      • response.addCookie(Cookie cookie);

    • 客户端获取Cookie:

      • request.getCookies();—没有拿特定Cookie的方法

    • 服务端会自动产生一个名称为JSEESSIONID的Cookie

  5. Session

    • 生成方式

      • 客户端第一次请求时,(匹配失败则)服务器端生成一个seesion,会自带一个唯一的sessionID,随后会生成一个JSEESSIONID的Cookie值相同,随响应传递给客户端,客户端与服务端的会话的对应是由JSEESSIONID的值和sessionID对应来实现的.客户端第n次请求时,不需要再创建session了,用JSEESSIONID找到session,如果找到,则匹配成功,说明以前访问过。

    • 和Cookie的不同

      • 存放位置不同
      • 安全性不同(Session保存在服务器端更安全些)
      • 保存的内容(Cookie内容只能存String而Session可以存任意对象

    • 方法

      • String getId()–获取sessionID

      • boolean isNew():判断是否是第一次访问

      • void invalidate():使session失效

      • void setMaxInactiveInterval(s):设置最大有效非活动时间

      • int getMaxInactiveInterval():获取最大有效非活动时间

int getMaxInactiveInterval():获取最大有效非活动时间

  1. application—全局对象

    • 方法

      • String getContextPath():项目的虚拟路径 网页访问的路径

      • String getRealPath(String name):获取应用任意虚拟路径对应的绝对文件路径

Java集合

Java

集合

问题

  1. 既然有数组了为什么还要有集合

    答:因为数组定长且只能存储一种类型的数据

  2. 为什么会有这么多的容器

    答:内部的数据存储结构不同

集合继承实现图

Collection接口

  • List和Set接口的父接口
  • List和Set接口的区别
    • List:元素可以重复,有索引
    • Set:元素无序且不能重复
    • 此处指的顺序是指存入和取出顺序不同
  • 要记得特有方法
    • obj.isEmpty();
    • obj.size();
    • obj.contains(Object o);
    • obj.retainAll(Collection<?> c):求交集并改变调用的集合
    • obj.add(E e)
    • obj.remove(Object o)
  • 迭代器(Iterator接口)
    • Iterator obj.iterator();
    • 每一个容器中都定义了一个迭代器用于操作容器本身

List接口

  • 集合的常用方法

    • 增:add(）
    • 删:remove()
    • 改:set()
    • 查:get(),subList()[对其操作会反馈到原集合],ListIterator()
    • 获取下标:indexOf()

  • List集合中引入了ListIterator迭代器

    • 因为原来的Iterator接口只提供了remove和next两种操作,为了避免并发修改异常在该迭代器中提供了增删改查操作,以及与next相反的previous操作
    • java.util.ConcurrentModificationException 并发修改异常 出现在使用迭代器的next的同时使用了集合的修改方法引发的异常

  • 引出的子类对象

    • ArrayList:数组结构,查找快增删慢,jdk1.2出现用于替代Vector,空间增长为50%增长,线程不同步

    • LinkedList:链表结构,查找慢,增删快,线程不同步

      • 特有方法(First/Last)
        • removeFirst/Last()[集合为空会出异常] —–pollFirst/Last()[jdk1.6]集合为空返回null
        • addFirst/Last()void —–offerFirst/Last()[jdk1.6]Boolean
        • getFirst/Last()[集合为空会出异常] —–peekFirst/Last()[jdk1.6]集合为空返回null

    • Vector:数组结构,jdk1.0出现,线程同步,空间100%增长

      • 特有方法(Element)
        • Enumeration obj.elements();
        • Enumeration 枚举类 其实就是Iterator前身,用法相同

  • 判断元素是否相同根据equals方法来确定

Set接口

  • 基本方法于Collection接口相同

  • 引出的子类对象

    • HashSet:哈希表结构,线程不同步,避免哈希冲突的方式类似于链地址法

      • 存储时先比较hashCode方法值是否相同,不同则存储,相同则比较equals判断,如果仍然相同,认为是统一对象,否则在hachCode值存储的位置下扩展链。
      • 比较对象是否相同先比较hashCode再比较equals,只有两者都相同才认为两对象相等。

    • TreeSet:红黑树结构(属于二叉排序树),线程不同步,有两种比较方式

      • 存储类实现java.lang.Comparable接口,实现compareTo方法,称为自然顺序,很多类都实现了(String,Integer)
      • 构造器类实现java.util.Comparator接口,实现compare方法,称为构造器,将构造器传入集合构造方法,实现排序
      • 如果上述两种方式都实现了以构造器为主
      • TreeSet标识元素相等的条件为compareTo方法或者compare方法的返回值(0)

    • LinkedHashMap

      • 继承HashMap实现Set接口

      • 底层Hash表加链表(记录元素的存储顺序),保证元素有序

Java泛型(jdk1.5)

  • 虽然集合可以存储任意对象带来了方便,但是因为类型不明确导致强制类型转换操作过多,且代码不明确,引入了泛型,泛型只能用于引用数据类型
  • ArrayList <String> list = new ArrayList<String>();
  • 泛型可以定义在类上,方法上,接口上
  • 注意:在静态方法中是不能访问定义在类上的泛型的,因为只有在定义对象时才能确定类上的泛型,而在此之前静态方法已经出现
  • this.<String>show();
  • 调用定义了泛型的方法为:在方法名前加<具体数据类型>,但是可以通过形式参数来推断,所以大多数时候可以省略
  • <?>代表任意类型,?代表统配符,之后又提出了泛型限定<? extends E>,<? super E>,前者在Collection中有应用,后者在Comparator接口中有应用。泛型限定用于对？的功能进行进一步的限定。
  • ArrayList <Object> list = new ArrayList<String>();
  • 上面这样的代码会出错,泛型并不支持这种多态,这样会导致声明为String的列表中有可能会存放其他Object的子类对象造成类型转换异常。

Map接口

  • 虽然Map接口并没有实现Collection接口但是仍然属于集合框架

  • Map<K,V>,用于存储键值对,其中键唯一,值不唯一

  • 常用方法

    • 1.添加:

      • V put(K key, V value):存相同的键会替代前键值对并返回原来的值,如果没有相同的键会返回null(与Set集合不同要注意)

    • 2.删除

      • clear()

      • remove(Object key)

    • 3.判断

      • containsKey(Object key)

      • isEmpty()

    • 4.获取

      • get(Object key)

      • size()

      • Set<V> values():获取值的Set集合

      • Map.Entry<K, V>[内部的静态接口] entrySet()

      • Set<K> keySet()

  • 常用的子类(所用比较的方法都是对键操作,与Set集合类似)

  • Hashtable

    • 底层是哈希表,不能存入null键或null值 jdk1.0 线程同步

  • HashMap

    • 底层也是哈希表,允许使用null键或null值 jdk1.2 线程不同步 效率高

  • TreeMap

    • 底层二叉树排序树结构,线程不同步,给map集合中的键进行排序

  • LinkedHashMap

    • 继承HashMap,实现Map接口

    • 底层Hash表加链表(记录元素的存储顺序),保证元素键有序

  • Set底层就是用了Map的相关实现方法

Utils接口

  • Collections类(里面全是静态方法)

    • 常用方法

    • 排序

      • <T extends Comparable<? super T>> void sort(List<T> list)
      • void sort(List<T> list,Comparator<? super T> c)

    • 最大最小

      • <T extends Object & Comparable<? super T>> T max(Collection<? extends T> coll)
      • <T> T max(Collection<? extends T> coll,Comparator<? super T> comp)
      • 为什么只有List有比较和最大值/最小值 因为Set有TreeSet,Map有TreeMap且它们都有相对应的First/Last方法

    • 查找

      • int binarySearch(List<? extends Comparable<? super T>> list,T key)
      • <T> int binarySearch(List<? extends T> list,T key,Comparator<? super T> c)
      • 为什么只有List有二分查找,因为只有它有脚标

    • 填充/替换/反转/交换/随机排序

      • boolean replaceAll(List<T> list,T oldVal,T newVal)
      • void reverse(List<?> list)
      • void swap(List<?> list,int i,int j)
      • void shuffle(List<?> list)
      • 如果要对列表的一部分填充的话可以用list中的subList对原列表进行填充

    • 获取逆向的比较器

      • <T> Comparator<T> reverseOrder():获取自然顺序逆序的比较器
      • <T> Comparator<T> reverseOrder(Comparator<T> cmp):获取指定比较器的逆向比较器

    • 获取线程同步的集合

      • <T> List<T> synchronizedList(List<T> list)
      • <T> Set<T> synchronizedSet(Set<T> s)
      • <K,V> Map<K,V> synchronizedMap(Map<K,V> m)

数组与集合的转换

  • Arrays类提供了许多数组操作的方法[数组转集合]

    • <T> List <T> asList(T... a):数组转换为List集合,

    • 原因在于可以方便去判断元素包含于集合[使用集合的方法来操作数组]

    • 生成的集合是定长的且改变会反馈到数组

    • 如果数组中的元素是基本数据类型,则会直接把数组本身存入到集合中

  • Collection类中的toArray方法[集合变数组]

    • Object[] toArray()

    • <T> T[] toArray(T[] a)

  • arrayList.toArray(new String[arrayList.size()])

    • 注意如果指定类型的数组长度小于集合的长度,那么方法内部会创建一个新的数组。长度为集合的长度

    • 如果如果指定类型的数组长度大于集合的长度,那么方法内部会使用传入的数组。多余的空间设置为null

    • 所以尽量指定一个大小刚好的数组,可以节省空间

Java反射

Java

反射

  1. 理解反射

    • 指的是在运行状态下,对于任意一个类,都可以知道这个类的所有属性和方法(包括构造方法);对于任意一个对象,都能调用它的任意一个方法和属性;这种动态获取的信息以及动态调用对象方法的功能称为Java的反射机制。

  2. 涉及的类

    • java.lang.Class

      • 概述

        • 程序经过javac.exe命令以后会生成一个或多个字节码文件(.class结尾) 接着我们使用java.exe命令对某个字节码文件进行解释运行。相当于将某个字节码文件加载到内存中,这个过程称为类的加载,加载到内存中的类称作运行时类,作为Class的一个实例。换句话说，Class的一个实例就对应一个运行时类
        • 在这个类中我们可以获取到类的域(Field),构造器(Constructor),方法(Method)

      • 创建Class类

        • final Class<?> getClass()–Object 要能创建对象才能用
        • 任何类型名.class 包含基本数据类型和数组
        • static Class<?> forName(String className)–Class最常用的方法因为本身传入的是字符串,可以使用变量而且可以和外部配置文件结合使用增强程序的扩展性。

      • 常用的方法

        • Field getField(String name)返回一个 Field 对象，它反映此 Class 对象所表示的类或接口的指定公共成员字段。可以拿到继承的域
        • Field getDeclaredField(String name)返回一个 Field 对象，该对象反映此 Class 对象所表示的类或接口所声明的所有字段。包括公共、保护、默认（包）访问和私有字段，但不包括继承的字段。
        • Constructor<T> getConstructor(Class<?>... parameterTypes)返回一个 Constructor 对象，它反映此 Class 对象所表示的类的指定公共构造方法。
        • Constructor<T> getDeclaredConstructor(Class<?>... parameterTypes)返回一个 Constructor 对象，它反映此 Class 对象所表示的指定类的所有的构造方法。
        • Method getMethod(String name,Class<?>... parameterTypes):返回一个 Method 对象，它反映此 Class 对象所表示的类或接口的指定公共成员方法。可以拿到继承的域
        • Method getDeclaredMethod(String name,Class<?>... parameterTypes):返回一个 Method 对象，该对象反映此 Class 对象所表示的类或接口的指定已声明方法。没有继承
        • T newInstance():创建此 Class 对象所表示的类的一个新实例。如同用一个带有一个空参数列表的 new 表达式实例化该类。

    • 后面的类都是java.util.reflex包中的

    • Field(域)

      • 常用方法
        • void set(Object obj,Object value):将指定对象变量上此 Field 对象表示的字段设置为指定的新值。如果为静态域就将obj设置为null。

    • Constructor(构造器)

      • 常用方法
        • T newInstance(Object... initargs):根据Class对象创建的构造函数来创建一个对象。

    • Method(方法)

      • 常用方法
        • Object invoke(Object obj,Object... args);根据Class对象创建的方法传入对象和实参进行调用。如果底层方法是静态的，那么可以忽略指定的 obj 参数。

    • AccessibleObject(取消权限检查)

      • 直接已知子类Constructor, Field, Method

      • 它提供了将反射的对象标记为在使用时取消默认 Java 语言访问控制检查的能力。对于公共成员、默认（打包）访问成员、受保护成员和私有成员，在分别使用 Field、Method 或 Constructor 对象来设置或获取字段、调用方法，或者创建和初始化类的新实例的时候，会执行访问检查。

      • 常用方法

        • void setAccessible(boolean flag):将此对象的 accessible 标志设置为指示的布尔值。值为 true 则指示反射的对象在使用时应该取消 Java 语言访问检查。值为 false 则指示反射的对象应该实施 Java 语言访问检查。
